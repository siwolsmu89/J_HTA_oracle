- 데이터 조작어
	- 테이블에 새 행 추가
	- 테이블에 기존 행 수정
	- 테이블에서 기존 행 삭제
	- INSERT 구문
		INSERT INTO 테이블명 [(컬럼명[, 컬럼명, 컬럼명, ...])]
		VALUES			(값, [,값, 값, ...])
		* INSERT INTO에서 컬럼명을 생략한 경우 테이블의 컬럼 순서에 맞게 값을 나열해야하고, 값을 생략할 수 없다.
			- 우리는 생략하지 않는다.(모든 값을 넣을 때에도 컬럼은 항상 적을 것)
		* INSERT INTO에서 컬럼명을 지정한 경우 지정된 컬럼의 개수와 순서에 맞게 VALUES의 값들을 나열해야 한다. 
		* 컬럼명을 지정하고 값을 입력한 경우 생략된 컬럼에 대해서는 DEFAULT로 설정된 기본값이 저장되거나 NULL값이 저장된다.
		* NULL 값을 저장하기
			- 컬럼을 나열할 때 해당 컬럼을 생략하면 자동으로 NULL값이 저장된다.
			- VALUES 절에서 값을 나열할 때 NULL을 저장하려는 자리에 명시적으로 NULL을 적어 준다.
		* 날짜값 저장하기
			- 시스템의 현재 날짜/시간 정보 저장 
				SYSDATE
			- 특정 날짜 및 시간값 저장
				'2020-04-25', '2020/04/25', '2020/04/25 12:23:31'
				TO_DATE('2020/04/25', 'YYYY/MM/DD'), TO-DATE('2020/03/31 14:01:33', 'YYYY/MM/DD HH24:MI:SS')
		* 일련번호 저장하기
			- 시퀀스로부터 일련번호를 인출해서 저장하기
				시퀀스명.NEXTVAL
			- 
		* 오류사항
			- NOT NULL로 지정된 컬럼의 값을 누락한 경우
			- 데이터 타입이 불일치하는 경우
			- 값의 길이가 컬럼에서 설정된 데이터 길이를 넘는 경우
			- 중복을 허용하지 않는 기본 키(PRIMARY KEY), 고유 키(UNIQUE) 컬럼에 중복된 값을 저장하려고 하는 경우
			- 외래 키(FOREIGN KEY) 제약 조건에 위배되는 값을 저장할 때(해당 컬럼에는 지정된 외부 컬럼에 이미 존재하는 값만 저장을 허락)
			- 체크 제약조건에 위배되는 값을 저장할 때('가격'처럼 0보다 작은 값이 없는 값, '학점'처럼 0~4.5의 범위가 있는 값 등은 체크 제약조건을 설정할 수 있음)
		* 다른 테이블에서 행을 복사해 저장하기
			INSERT INTO 대상테이블명 (컬럼명, 컬럼명, ...)
			SELECT 컬럼명, 컬럼명, ...
			FROM 원본테이블명
			[WHERE 조건식]
			* 대상 테이블의 컬럼명과 원본 테이블의 컬럼명이 일치할 필요는 없다.
			* 원본 테이블과 대상 테이블의 순서가 일치하는 컬럼들의 데이터 타입만 같으면 복사된다.
	- UPDATE 구문
		UPDATE 테이블명
		SET
			컬럼명 = 값[, 컬럼명 = 값, 컬럼명 = 값, ...]
		[WHERE 조건식]
		* WHERE 절을 적지 않으면 테이블의 모든 행들이 SET에 설정한 내용으로 바뀐다.
		* 특별한 경우가 아니면 WHERE 절을 사용해 조건식을 만족하는 행만 선택해서 값을 변경한다.
		* 서브쿼리를 사용해 획득한 값으로 컬럼의 값을 변경하는 것도 가능하다.
			UPDATE 컬럼명
			SET
				컬럼명 = (SELECT 컬럼명
					  FROM 테이블명
					  [WHERE 조건식])
			* 서브쿼리는 단일행 서브쿼리만 가능하다.
		* 오류 사항
			- INSERT 문의 오류 사항과 동일하다.
			- 자식 테이블에 자식 레코드(해당 컬럼의 값을 참조하는 행)가 존재하면 부모 테이블에서 해당 값을 변경할 수 없다.
			- 자식테이블에서 부모테이블의 값을 참조하는 컬럼의 값을 부모테이블에 존재하지 않는 값으로 변경할 수 없다.
	- DELETE 구문
		DELETE FROM 테이블명
		[WHERE 조건식]
		* WHERE절을 생략하면 테이블의 모든 행이 삭제된다.
		* 특별한 경우가 아니면 WHERE 절을 사용해서 조건식을 만족하는 행만 선택해서 삭제한다.
		* 오류 사항
			- 자식테이블에 자식레코드가 존재하는 행은 삭제할 수 없다.
	- MERGE 구문
		테이블에서 조건에 따라 데이터를 갱신하거나, 추가하는 기능을 제공한다.
		행이 존재하면 UPDATE를 수행하고, 새로운 행일 경우 INSERT를 수행한다.
		MERGE INTO 대상테이블 별칭
		USING 원본테이블 별칭
		ON (조인 조건식)
		WHEN MATCHED THEN
			UPDATE SET
			대상테이블컬럼명 = 원본테이블컬럼명,
			대상테이블컬럼명 = 원본테이블컬럼명,
			대상테이블컬럼명 = 원본테이블컬럼명
		WHEN NOT MATCHED THEN
			INSERT (대상컬럼명, 대상컬럼명, ...)
			VALUES (원본컬럼명, 원본컬럼명, ...)
